OVERVIEW
--------
This ComfyUI workflow restructures the 5-clip video generation pipeline from
5 independent subgraphs (15 model loads) to a batch-processing architecture
organized by model stage (3 model loads total).

By breaking open the LTX-2 subgraph internals and reorganizing node execution,
each model is loaded once and processes all 5 clips sequentially while resident
in VRAM, reducing redundant model I/O by 83%.

LAYER 3: Batch-Staged LTX-2 Expansion (Nodes 100-754)
  
  STAGE A - TEXT ENCODING (Nodes 100-104, 600-604)
    └─ 5× CLIP Text Encoders (100-104)
    └─ 5× Text Encoder Model Loaders (600-604)
    Process: All 5 cinematic prompts → CLIP embeddings
    Model residency: 1 text encoder load

  STAGE B - MODEL LOADING (Nodes 200-204, 250-254, 650-654)
    └─ 5× Checkpoint Loaders (200-204) - Load LTX-2 diffusion model
    └─ 5× Audio VAE Loaders (250-254) - Load audio encoding VAE
    └─ 5× LoRA Loaders (650-654) - Apply LoRA fine-tuning
    Model residency: 1 diffusion model + 1 audio VAE resident

  STAGE C - CONDITIONING & LATENT INITIALIZATION (Nodes 300-404, 450-454)
    └─ 5× LTXVConditioning (300-304) - Prepare positive/negative conditioning
    └─ 5× EmptyLTXVLatentVideo (350-354) - Initialize video latent space
    └─ 5× LTXVEmptyLatentAudio (400-404) - Initialize audio latent space
    └─ 5× LTXVConcatAVLatent (450-454) - Merge audio+video latents
    Process: Condition embeddings + empty latents → concatenated AV latents

  STAGE D - DIFFUSION SAMPLING (Nodes 110-114)
    └─ 5× SamplerCustomAdvanced (110-114)
    Process: All 5 clips undergo diffusion denoising sequentially
    Model residency: 1 diffusion model load (processes all 5)
    Key optimization: Diffusion model stays in memory for all 5 clips

  STAGE E - VAE DECODING (Nodes 500-504, 120-124, 700-704, 750-754)
    └─ 5× LTXVLatentUpsampler (500-504) - Upscale video latents
    └─ 5× VAEDecodeTiled (120-124) - Decode video latents → frames
    └─ 5× LTXVSeparateAVLatent (700-704) - Split sampled AV latents
    └─ 5× LTXVAudioVAEDecode (750-754) - Decode audio latents → PCM
    Process: All latents decoded to frames + audio while VAE resident
    Model residency: 1 VAE decode load (processes all 5)
    Key optimization: VAE stays in memory for all 5 decoding operations

LAYER 4: Final Aggregation (Nodes 140-141)
  • Node 140: DMM_VideoConcat - Stitch all 5 video clips together
  • Node 141: SaveVideo - Save final concatenated video

EXECUTION FLOW & NODE ORGANIZATION
-----------------------------------

1. DATA FLOW (Sequential):
   Data sources (1-6) → Cinematic prompts (10-14) → Video generation

2. BATCH TEXT ENCODING:
   Prompts (10-14) → CLIP Encoders (100-104) → Conditioning (300-304)
   [Model load 1: Text encoder]

3. BATCH DIFFUSION:
   Models (200-204) + LoRA (650-654) → Samplers (110-114)
   [Model load 2: Diffusion model]
   All 5 clips sample while diffusion model in VRAM

4. BATCH VAE DECODE:
   Latents → Upsample (500-504) → Decode (120-124) + Audio (750-754)
   [Model load 3: VAE]
   All 5 clips decode while VAE in VRAM

5. FINALIZATION:
   Decoded frames → CreateVideo (130-134) → Concat (140) → Save (141)

MODEL LOAD REDUCTION ANALYSIS
-----------------------------

Original Pipeline (5 independent subgraphs):
  • Clip 0: Load text encoder, diffusion, VAE
  • Clip 1: Load text encoder, diffusion, VAE
  • Clip 2: Load text encoder, diffusion, VAE
  • Clip 3: Load text encoder, diffusion, VAE
  • Clip 4: Load text encoder, diffusion, VAE
  = 15 model loads total (5 clips × 3 models each)

Optimized Pipeline (batch by stage):
  • Stage A: Load text encoder, encode all 5 clips, unload
  • Stage B: Load diffusion model
  • Stage D: Sample all 5 clips with diffusion resident
  • Stage E: Load VAE (often already in memory from Stage B)
  • Stage E: Decode all 5 clips with VAE resident
  = 3 model loads total (text encoder, diffusion, VAE)

VRAM EFFICIENCY GAIN: 83% reduction in model load operations
