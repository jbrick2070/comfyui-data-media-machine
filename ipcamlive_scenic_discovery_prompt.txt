Task: Discover live IPCamLive camera feeds in the Los Angeles metro area

I'm building an AI video generation pipeline called Data Media Machine that uses live webcam frames as conditioning images. I currently have 299 Caltrans highway cameras and only 3 verified scenic cameras from ipcamlive.com, all at the Weingart Center / Skid Row. I need to expand the scenic pool significantly.

Goal
Find live, working camera aliases on ipcamlive.com for feeds within the greater Los Angeles metro area. Aim for a broad geographic and visual mix — not just one type of camera.

Snapshot URLs use this format:
  https://g1.ipcamlive.com/player/snapshot.php?alias={ALIAS}

Verification rule
Verify each candidate camera by checking its snapshot URL.
Preferred method:
1. Send an HTTP HEAD request to the snapshot URL.
2. If it returns 200 and Content-Length > 20000, count it as live.
3. If HEAD is missing Content-Length, blocked, or looks unreliable, fall back to a normal GET request and measure the actual response body size.
4. Only include cameras that return HTTP 200 and an image payload greater than 20,000 bytes.
Anything smaller should be treated as a dead feed, placeholder, or error page.

Please include only aliases you directly verified today, not aliases inferred from page source alone.

Discovery methods to try
1. Google dork searches such as:
   - inurl:ipcamlive.com/player site:ipcamlive.com los angeles
   - site:ipcamlive.com/player ipcamlive santa monica
   - site:ipcamlive.com/player ipcamlive venice beach
2. Browse the public IPCamLive site and any location or directory pages:
   - https://www.ipcamlive.com
3. Search landmark + ipcamlive combinations:
   - Santa Monica Pier
   - Venice Beach
   - Malibu
   - Manhattan Beach
   - Griffith Observatory
   - Hollywood
   - Downtown Los Angeles
   - Marina del Rey
   - San Pedro
   - Long Beach
4. Check whether IPCamLive exposes a sitemap, public stream listing, embed pages, or other discoverable alias references.

Geographic filter
Keep cameras within about 40 miles of this center point:
  Latitude: 34.0622
  Longitude: -118.3437
This is Miracle Mile, Los Angeles.

If coordinates are available, calculate distance_miles using the haversine formula. If exact coordinates are not available, estimate from the location name as accurately as possible.

Output format
Return only a JSON array that I can merge directly into the "scenic" category of my camera_registry.json.

Each item must match this schema:
{
  "url": "https://g1.ipcamlive.com/player/snapshot.php?alias=THE_ALIAS",
  "label": "Human-readable location name",
  "priority": 1,
  "distance_miles": 0.0,
  "source": "ipcamlive",
  "verified_bytes": 0,
  "verified_date": "YYYY-MM-DD"
}

Field rules
- url: full snapshot URL using the verified alias
- label: concise human-readable location name
- priority: use 1 unless there is a reason to rank a camera lower
- distance_miles: haversine distance from 34.0622, -118.3437, or best estimate
- source: always "ipcamlive"
- verified_bytes: actual Content-Length from HEAD, or actual downloaded byte size from GET
- verified_date: date verified in YYYY-MM-DD format

Preferred camera types, in order
1. Beach / pier cams
   - Santa Monica, Venice, Malibu, Manhattan Beach
2. Skyline / cityscape cams
   - DTLA, Hollywood, observatory or hill views
3. Harbor / marina cams
   - Marina del Rey, San Pedro, Long Beach
4. Park / nature cams
   - Griffith, Runyon, mountain or canyon views
5. Street-level urban cams
   - Hollywood Blvd, Sunset Strip, Arts District, other recognizable public streetscapes

Exclude
Do not include:
- cameras outside the greater LA metro area
- cameras more than about 40 miles from 34.0622, -118.3437
- feeds returning less than 20 KB
- feeds requiring authentication or paid access
- private or residential cameras
- duplicate aliases
- unverified guesses

Final output requirement
Return only verified cameras as a JSON array, with no commentary before or after the JSON.
